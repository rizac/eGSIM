eGSIM API documentation

Communication with a web service requires an interface. A common example is a Graphical User Interface (GUI), which you can see by opening your browser, where you interact with the service using your mouse or keyboard on visual components.

A Web Application Programming Interface (API), on the other hand, lets you interact with the service through code using predefined URLs. For anything beyond simple data visualization, APIs are much more flexible and powerful than GUIs.

We recommend using the eGSIM API via Python and exploring our Jupyter Notebook examples, which show how to access the API through a single main function. For non-Python developers, users of external tools such as curl, or anyone needing more advanced configuration options, a brief eGSIM API overview is provided below.
